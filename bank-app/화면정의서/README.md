# 은행 코어뱅킹 시스템 — 페이즈별 프론트엔드 화면정의서

> 원본: `bank-app/은행_코어뱅킹_화면정의서_v2.md` (v2.0)
> 본 폴더는 원본 문서를 **프론트엔드 구현 관점**에서 페이즈별로 분해한 작업 가능 단위 화면정의서이다.

## 1. 문서 사용법

| 독자 | 읽는 순서 |
|---|---|
| FE 개발자 | `00_공통/` → 본인 페이즈 README → 화면별 .md |
| PM/기획 | 이 README → 각 페이즈 README |
| QA | 화면별 .md 의 "검증 시나리오" 섹션 |
| 컴플라이언스/감사 | `00_공통/00-02_법규매트릭스.md` + 화면별 "감사로그·법규" 섹션 |

각 화면 문서는 다음 13개 섹션을 표준으로 갖는다.

1. 메타 (URL · 권한 · 인증등급 · 적용법규 · 의존 화면)
2. 목적 / 사용자 시나리오
3. 화면 레이아웃 (텍스트 와이어프레임)
4. 컴포넌트 구성
5. 입력 항목 (필드명 · 타입 · 검증 · 마스킹 · UI 힌트)
6. 출력 항목 (필드명 · 포맷 · 마스킹)
7. 상태(state) 정의 (loading / empty / error / success / edge)
8. API 연동 명세 (모킹 기준 endpoint · method · payload · 응답)
9. 비즈니스 규칙
10. 보안·인증·감사로그 적재 포인트
11. 에이전트 연동 (위험도 등급별)
12. UX·접근성·반응형
13. 검증 시나리오 (단위 · 회귀)
